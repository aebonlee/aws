인공지능의 분류(AI vs. ML vs. DL)
  • AI(Artificial Inteligence)는 말 그대로 인간이 만드는(Artificial, 인공) 지능(Inteligence)입니다. 인간만이 가지고 있다고 여겨지는 지능을 구현하려는 시도를 전부 AI라고 부를 수 있습니다.
    • 사람이 아닌 것, 주로 기계나 컴퓨터가 사람처럼 ‘생각’하고 ‘행동’하는 것을 의미
    • 예: 자율주행 자동차의 보행자 인식과 정지 판단
  • 인간의 지성을 구현하는 방법은 다양합니다. 전문가 시스템(Expert systems)과 규칙 기반 시스템(Rule-based systems)이 대표적입니다. 카페나 음식점에서 주문할 때 키오스크를 쓰지는 않으셨나요? 주문에 필요한 규칙들은 비교적 간단해서 점점 더 많은 곳에 주문을 자동화하고 있습니다.
    • 예: 은행 대출 심사 초기 시스템
      if 연소득 > 5000만원 and 신용등급 > 800:
    대출_승인 = True
  • 머신러닝 (ML, Machine Learning)은 전문가 시스템, 규칙 기반 시스템과 마찬가지로 AI를 구현하는 방식 중 하나입니다. 머신(기계), 즉 컴퓨터가 경험을 통해 ‘학습’하는 것을 뜻합니다. 이때 경험은 데이터를 말합니다. 마치 여러분이 문제를 많이 풀면 실력이 늘어나는 것과 비슷합니다.
    • 컴퓨터가 데이터를 통해 '학습'하는 방법
    • 예: 스팸 메일 필터링, 신용카드 이상 거래 탐지
  • 딥러닝 (Deep Learning)은 머신러닝의 특별한 방식입니다. 인간의 뉴런 구조를 모방(신경망)하여 복잡한 패턴을 학습할 수 있습니다.
    • 사람의 뇌 구조를 모방한 방식 (사람 뇌처럼 생각하며 배우기)
    • 블랙박스 모델로 해석 가능성(Interpretability)이 가장 낮음
    • 예: 얼굴 인식, 음성 인식, 컴퓨터 비전, 자연어 처리(NLP), 시계열 예측(Time series forecasting, 예: DeepAR)
ML 분류
  • 학습 방식에 따른 ML 분류
    학습 방식설명종류와 예시
    지도 학습
    (Supervised learning)    		레이블이 달린 데이터로 학습
    입력 데이터와 해당 정답(레이블)이 주어진 상태에서 학습을 수행하는 방식    		1. 회귀(Regression): 매출, 집값, 온도 등 연속적인 수치 예측(선형 회귀, Linear Regression)
    - 독립 변수와 종속 변수 간의 관계를 모델링
    - 명시적인 가중치(weight) 파라미터 존재, 도메인 지식을 바탕으로 가중치 조정(adjust) 가능
    - 설명가능성(explainability)이 가장 높음
    2. 분류(Classification): 카테고리 예측
    - 심장 질환 위험을 예측하는 ML 모델을 개발 + 데이터셋에 심장 질환 유무(레이블)이 있는 경우
    - 이진 분류: 이메일이 스팸인지 아닌지 분류, 대출 신청 위험 평가(0 또는 1로 분류, Binary Classification, Logistic Regression)
    - 다중 분류(Multi-class classification): 3개 이상의 클래스로 분류(예: 동물을 개, 새, 고양이 등으로 분류. 리뷰를 긍정, 부정, 중립으로 분류)
    3. 의사 결정 트리(Decision Tree): 예/아니오 질문을 통해 답을 찾는 알고리즘
    - 스무고개(20 Questions)의 원리를 수학적으로 확장하여 자동화한 것
    - 회귀와 분류 모두 가능
    - 다중 클래스 분류 문제에도 활용 가능
    4. KNN(K-최근접 이웃, K-Nearest Neighbors): 새로운 데이터가 주어지면, 사전 정의된 카테고리(레이블) 중 가장 가까운 K개를 찾아 그 다수결로 분류
    비지도 학습
    (Unsupervised learning)    		레이블이 없는 데이터로 학습
    데이터의 패턴을 스스로 학습하여 그룹화하거나 이상치를 탐지하는 방식    		1. 군집화(K-means, Clustering): 데이터의 패턴을 분석하여 비슷한 데이터끼리 자동으로 그룹(클러스터)화
    - 세그멘테이션(segmentation): 고객의 인구 통계 및 구매 패턴을 기반으로 고객 그룹을 식별
    2. 차원 축소(Dimension reduction): 복잡한 정보를 간단하게 줄이기
    3. 군집 분석(Cluster analysis)
    4. 이상 탐지(Anomaly Detection): 무작위 절단 포레스트(RCF, Random Cut Forest), Autoencoder
    5. 밀도 추정(Density estimation)
    6. 자기 지도 학습(Self-supervised learning): 준지도 학습으로 분류되기도 함
    준지도 학습
    (Semi-supervised learning)    		정답이 있는 문제와 없는 문제를 섞어서 학습1. 사기 거래 찾기(Fraud identification)
    2. 감정 분석(Sentiment analysis)
    3. 문서 분류(Document classification)
    강화 학습
    (Reinforcement learning)    		보상 시스템을 통해 최적의 행동을 학습하는 방식
    시행 착오를 통해 학습    		1. 행동의 가치를 계산해 최고의 선택하기(예:미로에서 보상이 가장 큰 경로 찾기)
    2. 상황에 따른 최적의 행동 방침 학습하기(예:알파고가 바둑에서 최적의 수를 찾는 과정, 로봇의 길찾기)
    3. 행동을 선택하는 배우와 평가하는 비평가 조합(예: 게임에서 전략을 세우고 결과를 평가하며 개선하기)
    4. 환경 모델을 만들어 계획적으로 행동하기(예: 체스나 바둑에서 수를 미리 생각하고 두기)
    전이 학습
    (Transfer learning)    		사전 학습된 모델의 지식을 활용하여 새로운 관련 작업을 해결하는 학습
    개발 노력이 감소    		1. 강아지와 고양이를 구분하는 모델을 활용하여 다른 동물(예: 여우, 늑대)도 구별하는 모델로 확장
    2. 영어-프랑스어 번역 모델을 활용하여 영어-독일어 번역 모델을 개선
    3. 사전 학습된 딥러닝 모델을 사용하여 데이터셋에 대해 모델을 미세 조정
    • KNN vs. K-Means
      K-NN (K-최근접 이웃, K-Nearest Neighbors)K-means (군집화, Clustering)
      기존 그룹에 속하게 하기새로운 그룹을 만들기 (사전 정의된 그룹이 없는 경우)
      예제1: 특정 고객이 기존의 어느 고객 그룹에 속하는지 예측
      예제2: 새로운 전학생이 왔을 때 기존 그룹에 배정하기      		예제1: 고객의 인구 통계, 구매 패턴 등을 기반으로 비슷한 고객을 자동으로 분류
      예제2: 반 친구들을 비슷한 관심사(게임, 운동 등)를 보고 자동으로 그룹을 나눠줌
      기존에 그룹이 이미 존재해야 사용 가능
      기존 그룹: 학습 데이터에 정답(레이블)이 필요      		데이터에 레이블이 없고, 그룹을 사전에 모르는 경우에 사용.
      지도 학습비지도

      1. K-means (군집화, Clustering) - 비지도 학습

      • 데이터의 패턴을 분석하여 비슷한 데이터끼리 자동으로 그룹(클러스터) 을 만듦.
      • 데이터에 레이블이 없고, 그룹을 사전에 모르는 경우에 사용.
      • 예제: 고객의 구매 패턴을 기반으로 비슷한 고객을 자동으로 분류.

      1. K-NN (K-최근접 이웃, K-Nearest Neighbors) - 지도 학습

      • 학습 데이터에 정답(레이블)이 필요하고, 새로운 데이터가 주어질 때 기존 데이터와 비교 해서 분류.
      • 예제: 특정 고객이 기존의 어느 고객 그룹에 속하는지 예측.
  • 학습 아키텍처에 따른 ML 분류
    학습 방식설명
    중앙 집중식 학습
    (Centralized Learning)    		- 모든 데이터를 하나의 중앙 서버나 데이터 센터로 모아 모델을 학습시키는 전통적인 방식
    - 데이터 관리와 모델 훈련이 용이하지만, 데이터 전송 비용과 프라이버시 침해 위험이 존재
    - 가장 일반적인 방식
    연합 학습
    (Federated Learning)    		- 데이터를 중앙 서버로 보내지 않고, 각 사용자의 기기(로컬 환경)에서 모델을 학습
    - 이후 학습된 결과(모델 가중치)만을 중앙 서버로 보내 통합하여 전체 모델을 개선
    - 데이터 규정 준수와 개인정보 보호 보장 가능
    스플릿 학습
    (Split Learning)    		- 하나의 딥러닝 모델을 앞단(클라이언트)과 뒷단(서버)으로 나누어 분할 학습하는 방식
    - 로컬 장치에서는 입력 데이터 → 중간 표현까지만 계산 후 서버로 전달하고, 서버는 뒤쪽 모델을 학습
    - 원본 데이터가 외부로 나가지 않아 의료·금융 등 민감 데이터 협업 학습에 적합
    온라인 학습
    (Online Learning)    		- 새로운 데이터가 실시간으로 들어올 때마다 순차적으로 모델을 업데이트하는 방식
    - 데이터 전체를 다시 학습하지 않고 빠르게 반영할 수 있어 스트리밍 환경에 유리
    - 최신 패턴을 잘 반영할 수 있지만, 오래된 정보가 잊히거나 노이즈에 민감할 수 있음
  • 상황별 ML 학습 방법 선택 예시
    상황알고리즘
    유전자 특성에 따라 인간 유전자를 20가지 범주로 분류다중 클래스, 의사 결정 트리
    레이블이 지정되지 않은 고객 데이터를 보유 + 고객을 등급별로 분류하여 회사 제품을 광고하고 홍보비지도 학습(군집화)
    처음부터 새 모델을 만드는 것을 피하고, 사전 학습된 모델을 조정하여 새로운 모델을 만들길 희망전이학습
    들어오는 IP 주소를 분석하고 트래픽 패턴을 기반으로 IP 이상 탐지 시스템 개발
    특정 IP가 의심스러운 출처에서 온 것인지 확인 필요    		이상 탐지
    수작업 없이 사진 속 동물을 자동으로 식별하고 분류신경망(물체 감지)
    소프트웨어 개발 생산성을 높이기 위해 AI를 사용자연어 처리(NLP) 또는 LLM을 사용하여 코드를 생성
    제조 회사에서 AI를 사용하여 제품을 검사하고 손상이나 결함을 탐지컴퓨터 비전(신경망)
    교육용 게임에서 확률 예측 + 최소한의 운영 오버헤드
    (예: 빨간색 구슬 6개, 녹색 구슬 4개 등 여러 색이 있는 항아리에서 녹색 구슬 선택할 확률은?)    		간단한 규칙과 계산을 사용하여 확률 계산 코드 작성
    AI 모델을 만드는 것은 운영 오버헤드가 존재
    AI 모델을 훈련할 때 데이터 규정 준수와 개인정보 보호를 보장하는 ML 기법연합 학습(Federated Learning)

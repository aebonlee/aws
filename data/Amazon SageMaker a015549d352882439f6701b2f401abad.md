### 개요

- **인프라 관리 없이** 머신러닝 수명주기의 전 과정을 지원하는 **완전관리형** 서비스
    - 기계 학습의 시작과 끝(데이터 엔지니어링, 모델 구축, 훈련, 테스트, 튜닝, 배포)를 위한 통합 환경 제공
    - **자동화된 모델 튜닝, 평가** 등 ML 모델 개발을 위한 편의 기능 지원
- 머신러닝 수명 주기
    - 비즈니스 목표부터 모델 모니터링까지의 순환적 프로세스
        
        
        | 단계 | 설명 |
        | --- | --- |
        | 비즈니스 목표 식별 | - 비즈니스 목표와 성공 기준에 따라 비즈니스 가치를 측정
         - 모델이 특정 사용 사례에 부합하는지 평가하여 모델이 비즈니스 목표를 효과적으로 충족하는지 확인
         - 이해관계자 인터뷰를 수행하여 사용 사례를 구체화하고 측정 가능한 목표를 설정
        - 규정 준수 및 규제 요구 사항을 결정 |

          - 데이터 수집 및 준비
          - 데이터 전처리: 데이터 정제, 필터링, 변환, 정규화 등을 하는 단계(예: AWS Glue)
          - 탐색적 데이터 분석(Exploratory data analysis): 데이터의 특성과 패턴을 이해하기 위해 데이터를 분석하는 단계(예. 상관관계 행렬 생성, 통계 분석, 데이터 시각화 등). 본격적인 모델 개발 전에 데이터를 깊이 있게 탐구하는 과정
          - 특성(feature) 엔지니어링: 데이터에서 새로운 특성(변수)을 만들거나 기존 특성을 변환하는 단계 |

        - **IaC(Infrastructure as Code)** 활용**:** 코드를 통해 인프라를 관리. **확장 가능하고 일관된(scalable and consistent)** ML 워크로드를 위한 배포 간소화(streamline) |

        - 추론(예. 물체 감지를 위한 모델 배포 후 모델이 **새 이미지**를 분석하여 물체 식별) |
        | 모니터링 | - 조기 감지와 (문제) 완화를 통해 모델이 원하는 수준의 성능을 유지하고 있는지 확인
          - 예: 정의된 임계값을 초과하는 데이터 드리프트 발견 → 새로운 데이터로 모델 재학습

     - Feature attribution drift (특성 기여도 변화) - **Clarify 기능 활용** |  |

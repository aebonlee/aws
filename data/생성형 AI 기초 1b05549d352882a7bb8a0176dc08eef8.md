### 생성형 AI의 분류

- 생성형 AI(Generative AI, Gen AI)는 새로운 콘텐츠를 만들어낼 수 있는 모든 AI 시스템을 말합니다. 합성 데이터(synthetic data, 새롭게 만들어 낸 데이터)를 생성하는 데에도 활용합니다. 이는 다음과 같이 분류됩니다.
    - 1. 기초 모델(Foundation Model, FM)
        - 대규모 데이터(예. GPT-4의 경우 250p 책 약 20억권을 학습)로 사전 학습되어, 다양한 하위 작업(downstream task)에 응용될 수 있는 범용적인 AI 모델
        - 주요 특징
            - 높은 적응성(High Adaptability) 및 범용성: 하나의 모델이 별도의 대규모 재학습 없이 요약, 번역, 창작 등 다양한 작업에 쉽게 적응
            - 전이 학습(Transfer Learning): 특정 영역에서 학습한 지식을 새로운 작업이나 데이터에 효과적으로 전이하여 활용
        - 기초 모델을 ‘기초(Foundation)’ 모델이라고 부르는 이유
            - 전통적인 AI: 하나의 특정 작업(예: 번역)만 수행하도록 개발
            - 기초 모델: 별도의 재학습 없이 프롬프트 변경이나 약간의 파인튜닝만으로 요약, 창작, 질의응답 등 **다양한 작업에 쉽게 적응. 다양한 AI 모델과 애플리케이션을 구축하는 데 '기초(Foundation)'**가 됨
                - 텍스트 완성(Text completion) 모델, 지시 따르기(Instruction following) 모델, 텍스트 임베딩(Text embeddings) 모델, 이미지 생성(Image generation) 모델 등
        - 자기 지도 학습(Self-supervised learning)과 **대규모** **사전 학습**(Pre-trained)을 결합
            - 레이블이 지정되지 않은 방대한 데이터로부터 스스로 학습하여, 언어와 이미지에 대한 근본적인 이해와 패턴을 내재화. 지도 학습과 비지도 학습의 중간 형태
            - 마치 백과사전처럼 엄청난 양의 정보를 가지고 있는 AI(예. GPT, BERT)
        - 대표적인 예시
            - LLM(거대 언어 모델, Large Language Model): 텍스트 특화 모델
                - 주로 트랜스포머 구조 사용(Generative pre-trained transformers, GPT, **OpenAI 제품 모델 지칭 아님**)
                    - 핵심 원리: **셀프 어텐션(self-attention)** 메커니즘
                        - 문장 내 모든 단어의 관계를 동시에 계산하여(병렬 처리), 어떤 단어가 서로에게 중요한 영향을 미치는지 파악
                        - 문장 전체의 **맥락적 관계(contextual relationships)**를 효과적으로 학습
                    - 예. 자연어 처리(NLP, 감정분석, 번역, 챗봇 등), 자연어 입력을 SQL 쿼리로 **변환**하여 데이터베이스 분석 지원
                - 양방향(Bidirectional) 문맥 이해를 위한 트랜스포머 기반 모델도 존재(**Bidirectional** Encoder Representations from **Transformers**, BERT)
                    - 예. 문서에서 누락된 단어를 채움. 누락된 단어의 앞뒤를 파악해 문맥을 이해하여 단어 유추
            - 확산 모델(Diffussion Model): 이미지 생성에 특화
                - 순수 노이즈/무작위 데이터에서 시작해 의미 있는 정보를 점진적으로 추가
                - StableDiffusion이 대표적
            - 멀티 모달(Multi Modal): 다중 양식(텍스트, 이미지 등) 처리
                - Modal: 모델이 처리/생성하는 데이터 유형(텍스트, 이미지, 오디오 등)
    - 2. 기타 생성 모델 (전통적인 생성 모델 방식)
        - GAN(생성적 적대 신경망): 생성기와 판별기의 경쟁 학습. 생성기가 판별기를 속일 수 있는 데이터(주로 **이미지**) 생성 시도
        - VAE(변분 오토인코더): 데이터의 잠재 공간을 활용. **이미지** 생성/압축용

### 생성형 AI의 주요 개념

- 생성형 AI 안에 기초 모델(FM)이 있고, 기초 모델 안에 LLM이 있음
- LLM의 주요 용어
    
    
    | 용어 | 설명 |
    | --- | --- |
    | Token | 1. LLM이 텍스트를 처리하는 기본 단위
    2. 단어, 부분 단어, 문자 등이 될 수 있음
     - 예. "안녕하세요"라는 단어는 "안녕", "하세", "요"로 분리될 수 있음
    3. 과금(pricing)의 단위로도 사용
     - 모델 사용 비용은 입력 토큰과 출력 토큰 수에 기반하여 계산됨
     - LLM 사용 비용을 줄이고 싶은 경우, 제일 먼저 토큰의 수를 줄이는 것을 먼저 고려
     - 모델마다 토큰당 비용이 다름(예: Claude 3 Haiku는 입력 토큰 $0.25/백만 토큰, 출력 토큰 $1.25/백만 토큰)
     - 긴 프롬프트나 대화 기록은 토큰 사용량을 증가시켜 비용이 증가할 수 있음
     - 토큰 한도를 초과하면 일부 텍스트가 무시되거나 오류가 발생할 수 있음 |
    | Context Window | 1. LLM이 한 번에 처리할 수 있는 최대 텍스트 길이(토큰 한도)
     - 단일 프롬프트에 담을 수 있는 정보량
     - 입력 토큰이 모델의 컨텍스트 크기를 초과하면
      - 초과된 입력이 잘리거나  
      - 오류가 발생해 모델이 아예 응답을 생성하지 못함 |
    | SLM
    (Small Language Model) | 1. 파라미터 수가 적고, 학습 데이터셋이 작거나 모델의 구조가 단순한 모델
     - 크기가 작아 큰 모델(GPT, Claude 등)에 비해 언어 이해나 생성 능력이 떨어짐
     - 단순한 자연어 처리나 제한된 도메인에서 주로 사용
    2. 그대신 제한된 컴퓨팅 자원(스마트폰, PC, 엣지 디바이스)에서도 운영될 수 있고, 빠른 학습과 추론 속도를 제공
    3. 더 세분화해서 SLM 외에도 sLLM(smaller Large Language Model)으로 용어를 구분해서 쓰기도 하지만 자격증 수준에서는 구분하지 않아도 됨 |
    | Hallucination
    (환각) | 1. LLM이 그럴듯하고 사실처럼 들리지만 실제로는 잘못된 내용을 생성하는 현상 
     - 예1. 거짓된 역사적 사실을 진실처럼 서술 (세종대왕 맥북 프로 던짐 사건)
     - 예2. 마케팅 캠페인에서 존재하지 않는 제품 특징을 설명
     - 예3. 실제하지 않는 연구나 통계를 인용 |
    | Inference latency
    (추론 지연시간) | 1. 파운데이션 모델(FM)이 입력 요청을 처리하고 출력을 반환하는 데 걸리는 시간(속도)
    - 모델이 프롬프트를 받고 응답을 생성하기까지의 지연 시간
    2. 모델 선택 시 중요한 고려 요소
    - 실시간 챗봇: 낮은 지연 시간 필요
    - 배치 분석: 지연 시간보다 처리량(throughput)이 중요
    3. 혼동하기 쉬운 개념 비교
    - 모델 크기(Model size): 모델의 매개변수 수 → 지연 시간에 영향을 줄 수 있지만 속도 자체를 의미하지 않음
    - 컨텍스트 윈도우(Context window): 한 번에 수용 가능한 최대 입력 토큰 수 → 처리 용량이지 속도가 아님
    - 파인 튜닝(Fine-tuning): 모델을 특정 작업에 맞게 조정하는 프로세스 → 학습 과정이지 속도가 아님 |
    | Embeddings | 1. 텍스트, 이미지 등의 데이터를 수치적 표현(예. 벡터)으로 변환
     - 수치적 표현은 AI가 자연어를 이해하기 위한 수단
     - 고차원 데이터(텍스트, 이미지)가 저차원 밀집 데이터로 재표현
     - 이미지 내용을 분석하고 메타 데이터를 생성하는 데에도 사용
    2. 임베딩된 데이터는 축소된 차원 공간의 수치로 표현됨
      - **의미적 관계를 포착**: 의미적으로 유사한 데이터들은 비슷한 수치(예. 벡터 공간의 좌표)로 변환
      - 데이터(텍스트, 이미지)를 수학적으로 비교할 수 있는 능력을 제공
      - 이 좌표를 통해 벡터 검색이 가능해 짐 |
- FM의 학습 단계
    - 파운데이션 모델은 크게 두 단계를 거쳐 완성됨
    - 1단계: 사전 훈련(Pre-training)
        - 모델 가중치를 **무작위로 초기화(random initialization)**한 상태에서 시작
        - 대규모 웹 데이터 등 방대한 비지도(unlabeled) 데이터 코퍼스를 사용
        - **언어 모델링 목적 함수(language modeling objective)**를 통해 "다음 단어 예측" 등의 패턴을 학습
        - 이 단계의 결과로 범용적인 언어 이해 능력을 갖춘 기초 모델이 생성됨
    - 2단계: 파인 튜닝(Fine-tuning)
        - **이미 사전 훈련된 모델**에서 시작 (가중치를 무작위로 초기화하지 않음)
        - 레이블이 지정된 소규모 데이터셋으로 특정 작업에 맞게 모델을 조정
    - 구분 포인트
        
        
        | 구분 | Pre-training | Fine-tuning |
        | --- | --- | --- |
        | 시작점 | 무작위 가중치 | 사전 훈련된 가중치 |
        | 데이터 | 대규모, 레이블 없음 | 소규모, 레이블 있음 |
        | 목적 | 범용 언어 이해 | 특정 작업 최적화 |
        | 비용 | 매우 높음 | 상대적으로 낮음 |
- FM의 주요 파라미터
    - 생성형 AI 안에 기초 모델(FM)이 있고, 기초 모델 안에 LLM과 Diffusion Model이 있음
    - Text Generation (LLM - GPT, BERT)
        - 모델의 응답은 비결정적(non-deterministic)
        - 온도(Temperature)

            </aside>
            
            - 응답의 창의성(creativity)과 다양성, 무작위성(randomness) 조절
            - 높을수록: 모델이 높은 확률의 단어뿐만 아니라 낮은 확률의 단어도 선택할 가능성이 커짐
                - 창의적이지만 불안정하거나 무관한 응답. story telling, poetry에 적합
                - 환각(근거 없는 정보)를 생성할 가능성이 증가
                - 하지만 너무 낮으면 단조롭고 반복적인 응답을 할 수도 있음
            - 낮을수록: 모델이 가장 확률이 높은 단어를 우선적으로 선택하여 더 결정론적(deterministic)인 응답을 생성
                - 답변의 일관성이 높아지고, 논리적으로 안정적인 결과를 제공
                - 사실 기반 질의응답(Q&A), 기술적 설명, 법률, 의료, 데이터 분석 등 **정확성이 중요한 작업에 적합**
                - 하지만 창의성이 제한될 수 있으며, 반복적이고 예상 가능한 응답을 할 가능성이 높음
        - Top P (Top K)
            - 다음 토큰 선택시 고려(sampling)할 확률 범위
            - 예. Top P = 0.9는 상위 90% 확률의 토큰만 고려
            - 개발 프로세스를 통한 Top P 이해하기
                
                <aside>
                💡
                
                Top P는 언어 모델이 다음에 올 단어나 구절을 선택할 때 얼마나 다양한 옵션을 고려할지 결정하는 핵심 파라미터입니다. 마치 작사가가 다음 가사를 쓸 때 여러 선택지 중에서 고르는 것과 비슷합니다.
                
                **가사 생성 시나리오**
                
                다음과 같은 가사의 시작 부분이 있다고 해 봅시다. 
                
                ```
                "머리부터 발끝까지"
                ```
                
                AI는 이어질 가사로 여러 선택지를 고려합니다.
                
                - ‘오로나민 C’ (확률 40%)
                - ‘핫 이슈’ (확률 30%)
                - ‘사랑스러워’ (확률 20%)
                - ‘밀리오레’ (확률 10%)
                
                **Top P 적용**
                
                Top P가 0.9로 설정된 경우 확률을 더했을 때 90%가 될 때까지의 선택지만 고려합니다. 즉, 40% + 30% + 20% = 90%이므로 처음 세 개의 옵션만 고려 대상이 됩니다. ‘밀리오레’는 제외됩니다.
                
                | Top P | 결과 |
                | --- | --- |
                | 0.4 | ‘오로나민 C’만 고려 |
                | 0.7 | ‘오로나민 C’와 ‘핫 이슈’만 고려 |
                | 0.9 | 상위 3개 옵션 고려 |
                | 1.0 | 모든 옵션 고려 |
                
                Top P 값이 높을수록 더 창의적이고 다양한 응답을 생성할 수 있습니다. 하지만 너무 높으면 부적절하거나 맥락에 맞지 않는 응답이 생성될 수 있습니다.
                
                이러한 Top P 설정은 챗봇 응답, 스토리 생성, 코드 자동 완성 등 다양한 분야에서 활용됩니다. 각 용도에 맞게 Top P 값을 조정하여 최적의 결과를 얻을 수 있습니다.

    - Image Generation (Diffusion Model)
        - Classifier-Free Guidance(CFG, prompt strength): 텍스트-이미지 생성 모델에서 프롬프트와 생성된 이미지 간의 정렬도를 조절
            - (예전 방식처럼) 이미지가 텍스트 프롬프트와 얼마나 잘 매칭되는지 판단하는 분류기(Classifier)에 의존하지 않는(free) 방식
            - 스케일 값이 높을수록 프롬프트를 더 충실하게 따르는 이미지 생성
            - 스케일 값이 낮을수록 프롬프트와의 연관성은 낮아지고 더 자유롭고 창의적인 이미지 생성
        - Generation step(diffusion steps): 노이즈 제거 반복 횟수를 제어. 스텝이 많을수록 더 상세하고 정교한 이미지 생성
